# 2) Что мы сделали по шагам

## Шаг 2–4: Поняли данные и сделали baseline-сегментацию

- Проверили качество данных.
- После очистки осталось 89,982 пользователя.
- Пропуски есть, но часть из них структурные (например seller-метрики пусты у тех, кто не seller).
- Нашли теханомалии (user_id, даты), учли это в пайплайне очистки.
- Посмотрели распределения и поведение.
- Сильная “длинная правая хвостовая” структура: есть небольшая группа очень активных пользователей.
- Это важный сигнал: “ядро” будет маленьким, но очень сильным.
- Задали два правила “Про-пользователя”.
  - strict (жестко, “элита”): 193 пользователя.
  - relaxed (шире, рабочий пул): 881 пользователь.

Почему два правила?  
Чтобы не зависеть от одной произвольной границы: strict для “чистоты сигнала”, relaxed для “операционной работы”.

## Шаг 5: Построили прозрачный рейтинг кандидатов (ProScore)

- Сделали скоринг по ключевым метрикам (объявления, вопросы, рейтинг, отзывы, скорость ответа, сделки и т.д.).
- Ранжировали пользователей и получили top-1000 (поведенческий ranking).
- Проверили устойчивость (sensitivity):
  - к порогам отбора — ядро почти не разваливается,
  - к весам метрик — состав меняется заметнее (это нормально: разные бизнес-приоритеты дают разный список).

## Шаг 6: Добавили текстовый/NLP-слой

- Мы не строили тяжелую NLP-модель.
- Использовали понятные маркеры:
  - professionalism,
  - charisma,
  - expertise.
- Смешали:
  - behavior score (поведение),
  - style score (текст/коммуникация).
- Базовый blend: 70% behavior + 30% style.
- Получили обновленный top-1000 для пилота:
  - он не полностью новый, а частично пересекается с поведенческим ядром,
  - значит стиль действительно добавил полезный сигнал.

## Шаг 7: Выбрали категорию для старта пилота

Сравнили 4 категории из задания: Авто, Недвижимость, Электроника, Хобби и отдых.  
Для каждой считали:

- concentration_idx (насколько категория “перепредставлена” в ядре),
- engagement_idx (насколько сильна по вовлечению),
- priority_score = concentration_idx * engagement_idx.

В финальном step6-сценарии:

- Электроника: priority 1.409
- Авто: 1.394
- Недвижимость: 1.206
- Хобби: 0.972

Решение:

- Pilot: Электроника
- Fallback: Авто

# 3) Почему получились именно такие результаты

Почему “ядро” маленькое?  
Потому что “про-пользователь” = пересечение нескольких сильных условий сразу (часто продает + быстро + много вопросов + высокий рейтинг + отзывы). Такое всегда резко сужает выборку.

Почему Электроника победила, хотя Авто очень близко?  
Потому что у Электроники в финальном сценарии:

- чуть лучше интегральный приоритет,
- хорошая концентрация,
- хорошее вовлечение,
- и главное — решение устойчиво по нескольким сценариям (она чаще лидер).

Почему Авто всё равно зафиксирован fallback?  
Потому что разрыв маленький, и по логике пилота это идеальный “план Б”: почти такой же сильный сегмент, если в пилоте Электроники KPI не добираются.

Почему top-1000, а не сразу больше?  
Это управляемый размер для первой волны outreach и проверки гипотез.  
Сначала доказываем unit economics пилота, потом масштабируем.
